# candidate-interview-infra

This OpenTofu configuration provisions a single DigitalOcean VM (Droplet) with a raw block storage volume attached, designed for conducting job candidate interviews.

## Architecture

- **Interview VM**: A single DigitalOcean Droplet sized for interactive interview sessions
- **Block Storage**: An `ext4`-formatted volume attached to the VM for persistent data
- **No managed services**: Everything runs on the VM itself

## Prerequisites

1. **DigitalOcean Account**: Create an account and generate an API token
2. **SSH Key**: Upload your SSH public key to DigitalOcean and note the key ID
3. **OpenTofu**: Install [OpenTofu](https://opentofu.org/docs/intro/install/)

## Quick Start

1. **Set Environment Variables**:
   ```bash
   export DIGITALOCEAN_TOKEN="your-do-token-here"
   ```

2. **Configure Variables**:
   Create a `terraform.tfvars` file:
   ```hcl
   region         = "nyc1"
   droplet_name   = "interview-vm"
   ssh_key_ids    = ["your-ssh-key-id"]
   droplet_size   = "s-2vcpu-4gb"
   volume_size_gb = 50
   ```

3. **Deploy the Infrastructure**:
   ```bash
   tofu init
   tofu plan
   tofu apply
   ```

4. **Get Output Information**:
   ```bash
   tofu output
   ```

## Configuration Files

### Core Files
- `main.tf`: Droplet, block storage volume, and volume attachment resources
- `variables.tf`: Input variables for configuration
- `outputs.tf`: Output values including IP addresses and resource IDs
- `providers.tf`: DigitalOcean provider configuration

## Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `region` | DigitalOcean region | `nyc1` |
| `droplet_name` | Name of the interview VM droplet | `interview-vm` |
| `droplet_image` | OS image slug for the droplet | `ubuntu-22-04-x64` |
| `droplet_size` | Droplet size slug (CPU/RAM) | `s-2vcpu-4gb` |
| `volume_size_gb` | Size in GB of the block storage volume | `50` |
| `ssh_key_ids` | List of SSH key IDs to add to the droplet | `[]` |
| `enable_monitoring` | Enable DigitalOcean droplet monitoring | `true` |
| `tags` | List of tags to apply to all resources | `["interview"]` |

## Outputs

- `droplet_id` — ID of the interview VM droplet
- `droplet_ipv4_address` — Public IPv4 address of the interview VM
- `droplet_private_ip` — Private IPv4 address of the interview VM
- `volume_id` — ID of the block storage volume
